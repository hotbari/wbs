# Impeccable Design Context

## Design Context

### Users
Multi-role internal platform used daily by:
- **HR / Resource Managers** — primary users, allocate employees to projects, track capacity, manage staffing plans
- **Project Managers** — request staff, check availability, monitor allocation status per project
- **Admins** — configure the system, manage permissions, view org-wide dashboards

All roles are internal users working in a professional Korean corporate context. The platform is a core operational tool, not a marketing surface — users return to it daily under real work pressure. The UI must reduce cognitive load, not add to it.

### Brand Personality
**Structured · Trustworthy · Modern**

The interface should feel like a well-designed enterprise tool that people actually enjoy using — serious without being sterile, modern without being trendy. Think: the reliability of a good spreadsheet, the clarity of a great dashboard, the feel of software built with craft.

Emotional goals:
- **Confidence** — users should feel in control of complex allocation data
- **Clarity** — density is fine if hierarchy is clear
- **Trust** — data-heavy UI must never feel chaotic or ambiguous

### Aesthetic Direction
- **Visual tone:** Clinical-warm. Clean, airy, structured. Never cold or corporate-grey.
- **Color:** Zinc neutrals (warm grey) + Emerald accent. Emerald signals growth and positive allocation health.
- **Typography:** Plus Jakarta Sans (headings, UI) + GeistMono (numbers, percentages, dates). Tight letter-spacing on headings.
- **Cards:** Double-bezel elevation — outer shell + inner white core with inset highlight. Premium tactile quality.
- **Motion:** Spring physics on all interactions. Fast (stiffness 400–500, damping 28–35). Motion is purposeful, never theatrical.
- **Nav:** Floating glass pill island. Never a full-width sticky bar.
- **Density:** 4/10 — gallery-airy. Generous whitespace, not jam-packed.
- **Theme:** Light-first. Dark mode is supported but secondary.

**Anti-references (what to avoid):**
- Cold enterprise grey (SAP, legacy HRMS)
- Neon glows or oversaturated colors
- Symmetrical 3-column card grids (boring)
- Heavy shadows or drop-shadow overuse
- Decorative elements that don't carry meaning
- Emoji in UI text

**Tech constraints:**
- Framer Motion v12 for all animation
- Tailwind CSS v4 (no arbitrary values where variables exist)
- Phosphor Icons (@phosphor-icons/react) — duotone weight preferred
- CSS custom properties for all tokens (no hardcoded hex values in components)

### Design Principles

1. **Structure earns trust.** Every element must justify its position through visual hierarchy. If something doesn't guide the eye toward the next action, remove it.

2. **Motion is weight, not decoration.** Spring physics makes interactions feel physical and intentional. Animate only transform + opacity. Never animate layout properties.

3. **Density through clarity, not compression.** Show more by making each element more legible — tighter type scale, better hierarchy — not by shrinking everything.

4. **Emerald means health.** The accent color carries semantic meaning across the domain (allocation status, positive states, success). Use it sparingly so it retains signal value.

5. **Every role deserves the same craft.** Admin views, PM views, and HR views all get the same design quality. No "internal admin ugliness" accepted.
